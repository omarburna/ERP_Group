# Supabase Setup Guide

This guide will help you set up Supabase as the backend for your HR Job Application System.

## Step 1: Create a Supabase Project

1. Go to https://supabase.com
2. Click "Start your project"
3. Sign up or log in to your account
4. Click "New Project"
5. Fill in the project details:
   - **Name**: hr-job-system (or any name you prefer)
   - **Database Password**: Create a strong password (save this!)
   - **Region**: Choose a region close to you (e.g., West Africa for Ghana)
6. Wait for the project to be created (this may take 1-2 minutes)

## Step 2: Get Your Supabase Credentials

1. Once your project is ready, go to the **Settings** tab
2. Click on **API** in the left sidebar
3. Copy the following:
   - **Project URL** (looks like: https://xxxxxxxx.supabase.co)
   - **anon public** key (looks like: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...)

## Step 3: Configure the HTML File

1. Open `hr-system.html` in a text editor
2. Find these lines (around line 903-904):
   ```javascript
   const SUPABASE_URL = 'YOUR_SUPABASE_URL_HERE';
   const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY_HERE';
   ```
3. Replace them with your actual credentials:
   ```javascript
   const SUPABASE_URL = 'https://your-project-id.supabase.co';
   const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
   ```

## Step 4: Create Database Tables

Go to your Supabase project dashboard and follow these steps:

### Create the "jobs" table

1. Click on **SQL Editor** in the left sidebar
2. Click "New Query"
3. Paste and run this SQL:

```sql
-- Create jobs table
CREATE TABLE jobs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT NOT NULL,
    requirements TEXT NOT NULL,
    location TEXT NOT NULL,
    salary TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security (optional, for production)
ALTER TABLE jobs ENABLE ROW LEVEL SECURITY;

-- Allow public read access
CREATE POLICY "Public read access for jobs" ON jobs
    FOR SELECT USING (true);

-- Allow public insert access (for admin posting jobs)
CREATE POLICY "Public insert access for jobs" ON jobs
    FOR INSERT WITH CHECK (true);
```

### Create the "applications" table

1. Click "New Query" again
2. Paste and run this SQL:

```sql
-- Create applications table
CREATE TABLE applications (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    jobId BIGINT,
    jobTitle TEXT NOT NULL,
    applicantName TEXT NOT NULL,
    email TEXT NOT NULL,
    phone TEXT NOT NULL,
    resume TEXT NOT NULL,
    coverLetter TEXT,
    status TEXT NOT NULL DEFAULT 'received',
    interviewDate TIMESTAMP WITH TIME ZONE,
    interviewNotes TEXT,
    offerDetails TEXT,
    onboardingDate TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE applications ENABLE ROW LEVEL SECURITY;

-- Allow public read access
CREATE POLICY "Public read access for applications" ON applications
    FOR SELECT USING (true);

-- Allow public insert access (for job applications)
CREATE POLICY "Public insert access for applications" ON applications
    FOR INSERT WITH CHECK (true);

-- Allow public update access (for admin workflow)
CREATE POLICY "Public update access for applications" ON applications
    FOR UPDATE USING (true);
```

## Step 5: Test the Connection

1. Save the `hr-system.html` file
2. Open it in your browser
3. You should see the application load with data from Supabase
4. Try posting a job or submitting an application to verify it's working

## Troubleshooting

### "Supabase not configured" error
- Make sure you replaced the placeholder credentials in the HTML file
- Check that your Supabase project is active (not paused)

### "Error fetching data" error
- Check your browser console (F12) for detailed error messages
- Verify your Supabase URL and anon key are correct
- Make sure the tables were created successfully

### Tables not showing data
- Go to the **Table Editor** in Supabase dashboard
- Check if the "jobs" and "applications" tables exist
- If tables are empty, the app will automatically insert sample data

### CORS errors
- Go to **Project Settings** → **API**
- Add your localhost URL to CORS settings if needed (usually not required for anon key)

## Security Notes

For production use, consider:
1. Implementing proper authentication (Supabase Auth)
2. Restricting table access with more specific RLS policies
3. Using service role keys instead of anon keys for sensitive operations
4. Adding input validation on the backend
